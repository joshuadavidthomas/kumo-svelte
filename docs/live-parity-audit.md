# Live Kumo parity audit

Objective: compare every Svelte component page and demo against the live upstream Kumo docs at `https://kumo-ui.com` and the checked-in upstream source under `reference/cloudflare-kumo/`. Track behavior and styling evidence here as work proceeds.

## Audit method

For each component:

1. Read the upstream docs page/source under `reference/cloudflare-kumo/packages/kumo-docs-astro/src/pages/components/` and `reference/cloudflare-kumo/packages/kumo-docs-astro/src/components/demos/`.
2. Read the upstream component source under `reference/cloudflare-kumo/packages/kumo/src/components/`.
3. Open the upstream live page with `agent-browser`.
4. Open the local page with `agent-browser`.
5. Compare demo coverage, visible styling, computed dimensions/classes where useful, and interactive behavior.
6. Fix Svelte drift or document an intentional Svelte-only difference.
7. Re-run relevant checks after fixes.

## Global findings

- 2026-05-20: Home/dashboard sluggishness was caused by all dashboard cards mounting immediately, including offscreen Loader/Skeleton animations. Kept the visible Button loading indicator for upstream parity, and lazy-mounted offscreen dashboard previews instead.
- 2026-05-21: Dev server docs routes under `(docs)` were serving the docs error page because absolute `import.meta.glob` keys were empty in dev. Switched docs layout loading globs to relative patterns so local browser testing works in dev and preview.
- 2026-05-21: Corrected the audit method after visual spot-check failures: home-grid component previews are part of parity, not just component detail pages. Breadcrumbs and Flow had been incorrectly marked pass based on detail-page checks; their home previews were fixed and re-verified against live home-grid rendering.
