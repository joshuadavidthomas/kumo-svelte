# Component Detail Pages Checklist

Use this checklist to audit each component detail page against the upstream Astro docs and the Svelte 5 port expectations.

For each page:

- Load the SvelteKit page in `agent-browser`.
- Check browser console logs and page errors.
- Compare layout, content, examples, API tables, and visual styling against upstream docs.
- Fix missing or incorrect Svelte examples.
- Verify interactive demos behave correctly.
- Re-run validation after fixes.
